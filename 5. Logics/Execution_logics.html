Execution Parameters Tab

Full parameter reference — logic, engine behaviour & validation

Project: MegaServe AI Trading Engine  |  Last Updated: 2026-04-22
Source Files: tabs/execution_parameters.py  ·  engine/backtester.py  ·  engine/monitoring.py  ·  engine/execution.py  ·  core/validators.py  ·  core/helper.py

Overview

The Execution Parameters Tab is divided into two groups that jointly control how and when every leg in the portfolio enters the market.

GroupPurposeFields
Execution Settings Broker-facing and system-level execution settings Product, Strategy Tag, If One or More Leg Fail, Leg Execution (hidden)
Execution Mode Controls when the portfolio is allowed to start entering legs (market-condition trigger) Portfolio Execution Mode, Based On, Entry Price, Rounding Value, Adjust Price
Timing Controls which days and what intraday/calendar windows the portfolio is active Run On Days, Start Time, End Time, SqOff Time

Together these 13 parameters (12 visible + 1 hidden) determine how, when, and on which days every leg in the portfolio enters and exits.

PART 1 — Execution Settings

1. Product execution_params.product str Default: MIS

Specifies the order product type sent to the broker for every leg in this portfolio.

Options

OptionFull NameUsage
MIS Margin Intraday Square-off Intraday only. Auto-squared by broker at EOD (~15:10–15:20). Lower margin requirement. Most common for options strategies.
NRML Normal / Carry-Forward Overnight / positional product. Full margin required. Use when Positional Portfolio is enabled.
Engine Note: The product field is passed directly to the broker order API. The backtester does not evaluate it — it is metadata forwarded to the live execution layer. CNC is not supported — options backtesting uses MIS or NRML only.
2. Strategy Tag execution_params.strategy_tag str Required — cannot be empty

Links this portfolio to a Strategy defined in the Strategies configuration. Every enabled portfolio must have a non-empty Strategy Tag before it can be run.

Engine Resolution

strategy = next(
    (s for s in strategies
     if s.get("strategy_tag") == strategy_tag or s.get("tag") == strategy_tag),
    None
)

This lookup is used to:

  • Identify which user accounts (brokers / API keys) should execute this portfolio's orders.
  • Validate the strategy is enabled — a disabled strategy raises a ValueError and blocks the run.
  • Associate trades with the correct backtester run entry: pid = user|strategy_tag|portfolio_name.

3. If One or More Leg Fail execution_params.leg_fail_action str Risk-Critical

Defines the recovery policy when one or more legs in a multi-leg portfolio fail to place (order rejection, no liquidity, strike not found).

OptionBehaviourUse When
KeepPlacedLegs (default) Legs placed before the failure remain open and active. Failed leg is logged and skipped. Partial fills are acceptable; e.g. hedge leg failure doesn't create naked risk on remaining legs.
CancelPlacedLegs All legs placed before the failure are cancelled / squared off. The entry attempt is rolled back to zero net position. Portfolio only makes sense as a complete unit — e.g. Iron Condor where missing 1 leg creates naked risk.
Backtester Note: The backtester does not simulate partial-fill failures. This setting has no effect during backtesting — it applies only in live/paper trading.
4. Leg Execution execution_params.leg_execution str Hidden — Backward Compat Only

This field is not visible in the current UI but is still saved and loaded for backward compatibility with older portfolio configs.

OptionBehaviour
Parallel (default)All legs submitted simultaneously to the broker.
SequentialLegs submitted one at a time, each waiting for previous confirmation.
The current codebase defaults to Parallel. The UI has removed this control from view. It remains in get_data() / set_data() and core/validators.py to avoid breaking existing saved configs.

PART 2 — Execution Mode

5. Portfolio Execution Mode execution_params.portfolio_execution_mode str Default: Start time

The master trigger that controls when the engine transitions from WAITING_FOR_ENTRY → ACTIVE, allowing all legs to begin entering.

State Machine

WAITING_FOR_ENTRY
Dynamic modes (initial state)
→
ACTIVE
Trigger condition met

Start time / Manual modes initialise directly to ACTIVE.

Mode Reference

5.1 Start time Default

Portfolio enters legs at exactly the Portfolio Start Time (set in the Timing tab). No additional condition is checked.

status = "ACTIVE"  # set immediately at state build time
  • Based On, Entry Price, Rounding Value, Adjust Price are all hidden in the UI.
  • The engine checks curr_time >= timing["start"] before firing any leg, but exec_mode itself is pre-set ACTIVE.

5.2 Manual

The portfolio waits indefinitely until it receives an external trigger signal — e.g. from another portfolio's "Start Other Portfolio" action on its Target/Stoploss hit.

status = "WAITING_FOR_ENTRY"
triggered by:  exec_mode["triggered"] = True  (set externally)
  • Used for portfolio chaining — Portfolio A's target fires "Start Other Portfolio B".
  • A triggered Manual portfolio is exempt from the end_blocked guard (can enter even late in the day).
  • Based On, Entry Price, Rounding Value, Adjust Price are hidden in the UI.

5.3 CombinedPremium

Waits until the ATM straddle premium (ATM Call mid + ATM Put mid) reaches the configured target level.

# ATM combined premium computed every tick:
premium = (ask_atm_call + bid_atm_call)/2 + (ask_atm_put + bid_atm_put)/2

# Trigger direction auto-detected:
trigger = curr_premium <= target   # when target < day_start
trigger = curr_premium >= target   # when target > day_start
trigger = True                      # when target == day_start
Based OnBehaviour
NoneUses absolute Entry Price as target. Fires when premium crosses that fixed level.
DayOpenCaptures premium at market open (first tick). Applies Rounding + Adjust Price to compute target. Fires when curr_comb < target.
StartTimeCaptures premium at Portfolio Start Time. Fires immediately if rounding > 0, no adjust string, or when curr_comb < target.

5.4 UnderlyingLevel

Waits until the underlying index price touches or crosses the configured target level.

# Based On = None: direction inferred from day-start
trigger = curr_idx <= target   # when target < day_start
trigger = curr_idx >= target   # when target > day_start

# Based On = DayOpen / StartTime: crossover model
trigger = (last_idx <= target <= curr_idx) or (last_idx >= target >= curr_idx)
  • Uses index OHLC close price per tick.
  • For DayOpen/StartTime: requires the index to actually cross the level, not just be above/below it at start.

5.5 CombinedPremiumCrossOver

Advanced version of CombinedPremium that requires a full crossover cycle before triggering, filtering premature entries.

DayOpen Crossover Sequence

STAGE_BELOW_INIT
Wait for premium to first drop below target
↓ premium < target
STAGE_ABOVE
Wait for premium to rise back above target
↓ premium ≥ target
STAGE_BELOW_TRIG
Wait for final decay below target
↓ premium < target
TRIGGERED → ACTIVE

StartTime Crossover Sequence

1. Base set at StartTime: WAIT_ABOVE (waiting for rise above target + 0.01)
2. Rise detected → WAIT_BELOW (waiting for decay below target)
3. curr_comb < target → TRIGGER
6. Based On execution_params.based_on str Default: None Hidden for Start time / Manual

Determines the reference point used to calculate the dynamic entry target level for CombinedPremium, UnderlyingLevel, and CombinedPremiumCrossOver modes.

OptionReference Capture PointAdditional Fields Shown
None No reference capture. Entry Price used directly as absolute target. Entry Price
DayOpen First tick of the trading day (r_idx == 0). Market-open price. Rounding Value, Adjust Price
StartTime First tick at or after Portfolio Start Time. More controlled — measures from strategy activation. Rounding Value, Adjust Price 
# Engine reference capture:
if exec_cfg["based_on"] == "DayOpen" and exec_cfg["ref_val"] is None:
    exec_cfg["ref_val"] = curr_ref_candidate   # at r_idx == 0

elif exec_cfg["based_on"] == "StartTime" and exec_cfg["ref_val"] is None:
    if curr_time >= timing["start"]:
        exec_cfg["ref_val"] = curr_ref_candidate
7. Entry Price execution_params.entry_price float Default: 0.0 Shown only when Based On = None

The absolute target level used when Based On = None. This is the exact premium or index price the system waits for before triggering portfolio entry.

Engine Assignment

# Direct copy to target_val — no rounding or adjustment applied
if exec_cfg["based_on"] == "None" and exec_cfg["target_val"] is None:
    exec_cfg["target_val"] = exec_cfg["entry_price"]

Save-time Validation

# In portfolio.py → save_portfolio()
if mode not in ["Start time", "Manual"] and based_on == "None":
    if entry_price == 0.0:
        QMessageBox.warning(...)  # blocks save — entry price required
Validation: Must be non-zero when mode is dynamic and Based On = None. The portfolio cannot be saved with entry_price = 0 in this configuration.
8. Rounding Value execution_params.rounding_value float Default: 0.0 Shown only when Based On = DayOpen or StartTime

Applied to the captured ref_val to snap it to the nearest grid before applying the adjustment. Prevents entry targets from landing at awkward fractional price levels.

Rounding Algorithm

if rnd > 0:
    if rnd < 1.0:
        # Fractional threshold: snap to floor or ceiling
        frac = ref % 1.0
        ref  = float(math.floor(ref) + 1) if frac >= rnd else float(math.floor(ref))
    else:
        # Integer rounding: snap to nearest multiple of rnd
        ref = math.floor(ref / rnd + 0.5) * rnd

9. Adjust Price execution_params.adjust_price str Default: "" (empty) Shown only when Based On = DayOpen or StartTime

A string expression applied to the (already rounded) reference value to compute the final entry target. Supports absolute and percentage adjustments.

Adjustment Algorithm

adj_str = exec_cfg["adjust"].strip()
target  = ref   # ref already rounded

if adj_str:
    if adj_str.endswith("%"):
        target = ref * (1 - float(adj_str[:-1]) / 100)
    else:
        target = ref + float(adj_str)
Adjust Priceref_valtarget_valFormula
+104820.04830.04820 + 10
-154820.04805.04820 + (−15)
+5%200.0190.0200 × (1 − 5/100)
-10%200.0220.0200 × (1 − (−10)/100)
empty4820.04820.0No adjustment
% Sign Note: The formula applies 1 − pct/100. A positive % string reduces the target; a negative % string increases it. This is designed for premium-decay scenarios (e.g. +5% on a 200-point straddle targets 190).

PART 3 — Timing

9. Run On Days timing.run_days str Default: All Days

Specifies which calendar days the portfolio is permitted to run. Acts as a coarse day-of-week gate applied before any intraday time-window or condition checks.

OptionDays ActiveUse Case
All DaysMon–Sun24/7 or weekend-eligible instruments
Mon - FriMon–FriStandard weekday-only options strategies
Mon - ThuMon–ThuStrategies that avoid Friday expiry volatility
CustomUser-selectedReveals 7 checkboxes (Mon–Sun) for arbitrary day selection

Data Storage

{
    "run_days":     "Mon - Fri",
    "selected_days": ["Mon","Tue","Wed","Thu","Fri"]   // only present when Custom
}
10. Start Time timing.start_time str (HH:mm:ss) Default: 09:30:00

The earliest intraday time the portfolio is allowed to enter legs. Until this time is reached no legs can fire, regardless of their individual leg-level start time overrides.

Engine Behaviour

_raw_ls  = parse_time_str(leg["conf"].get("Start Time"))   # leg-level override
if _raw_ls and timing.get("is_winter"):
    _raw_ls = add_one_hour(_raw_ls)                         # winter-time adjustment
leg_start = _raw_ls or timing["start"]                      # fallback to portfolio start

if curr_time >= leg_start:
    can_execute = True
  • Each leg can optionally carry its own Start Time override (set in Leg config).
  • If no leg-level override, the portfolio Start Time is used as the gate.
  • Winter/summer time adjustment is applied automatically via is_winter flag.
11. End Time timing.end_time str (HH:mm:ss) Default: 16:15:00

The cutoff time for new entries. After this time no new leg entries are initiated. Existing open positions continue to be monitored for Target/SL.

Engine Behaviour

if timing["end_blocked"] and not _triggered_manual:
    continue   # skip new entry — end time reached

Manual Portfolio Exception

A Manual portfolio that has been externally triggered bypasses the end_blocked guard, enabling late-session portfolio chaining.

_triggered_manual = (
    state["exec_mode"]["mode"] == "Manual"
    and state["exec_mode"].get("triggered")
)
if timing["end_blocked"] and not _triggered_manual:
    continue   # blocked — unless manually triggered
Constraint: End Time must be after Start Time. The system does not enforce this at the UI level — it is the user's responsibility.
12. SqOff Time timing.sqoff_time str (HH:mm:ss) Default: 16:15:00

The intraday forced exit time. At or after this time, all active legs not already exited via Target or Stoploss are force-squared-off at the last known market price.

Typical Intraday Pattern

09:30:00  Start Time  →  Entry gates open
16:15:00  End Time    →  No new entries
16:15:00  SqOff Time  →  Force-close all remaining positions
End Time and SqOff Time are commonly set to the same value for standard intraday strategies. They can differ if you want to stop new entries at an earlier time but allow Target/SL management to continue until a later forced close.

PART 5 — Engine Initialisation

_build_exec_mode() in engine/backtester.py constructs the runtime state dict from saved config:

def _build_exec_mode(ep: dict) -> dict:
    mode = ep.get("portfolio_execution_mode", "Start time")
    return {
        "mode":            mode,
        "based_on":        ep.get("based_on", "None"),
        "entry_price":     float(ep.get("entry_price", 0.0)),
        "rounding":        float(ep.get("rounding_value", 0.0)),
        "adjust":          str(ep.get("adjust_price", "")),
        "status":          "WAITING_FOR_ENTRY" if mode not in ("Start time", "Manual") else "ACTIVE",
        "ref_val":         None,    # set on DayOpen/StartTime first tick
        "target_val":      None,    # computed from ref_val + rounding + adjust
        "day_start_val":   None,    # captured at r_idx == 0 for logging
        "triggered":       False,
        "crossover_state": "",      # used by CombinedPremiumCrossOver
    }
Key: Start time and Manual initialise directly to ACTIVE. All other modes start WAITING_FOR_ENTRY and rely on check_portfolio_exec_mode() running every tick until triggered.

PART 6 — Entry Gating in execution.py

Inside entry_fill_leg(), the exec_mode state gates every attempted leg entry:

exec_mode = state["exec_mode"]["mode"]

if exec_mode == "Manual" and not state["exec_mode"].get("triggered"):
    return   # skip — not yet externally triggered

elif exec_mode not in ("Start time", "Manual"):
    if state["exec_mode"]["status"] == "ACTIVE" and state["exec_mode"]["triggered"]:
        pass   # allow entry
    else:
        return   # skip — still WAITING_FOR_ENTRY

elif state["exec_mode"]["mode"] == "Manual" and state["exec_mode"].get("triggered"):
    pass   # allow entry (externally triggered)
A Manual portfolio triggered late in the day (after end_time) is still allowed to enter — the end_blocked guard is bypassed for triggered Manual portfolios.
