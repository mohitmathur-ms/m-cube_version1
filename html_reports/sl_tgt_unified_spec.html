SL & Target — Unified Multi-Asset-Class Spec

Leg-level · Portfolio-level · User-level features and applicability across IF, FX, CF, XF

Supersedes sl_tgt_feature_applicability.md and sl_tgt_complete_spec.md

Currency rule: INR ₹ for Indian Index Futures only. USD $ for everything else (US Index Futures, FX, Commodity, Crypto).
Per-asset-class customization: Most features run uniformly across IF, FX, CF, XF. A small number are asset-class specific — e.g., Underlying-based SL/Target types apply to Index Futures only and do not apply to FX, Commodity, or Crypto. The applicability matrix in each section shows where each feature applies.

Decisions Locked

D1 · Currency rule Locked

INR (₹) for Indian Index Futures, USD ($) for everything else; UI symbol matches per-instrument metadata.

D2 · "SameStrike" → "Same Contract" Locked

Engine + UI rename. Futures use contract month, not strike, so the action ReExecute SameStrike at EntryPrice becomes ReExecute Same Contract at EntryPrice.

D3 · Testing default n_lots = 1 Locked

lot_size sourced from per-instrument metadata.

D4 · Three-level structure preserved Locked

User, Portfolio, Leg — across all four asset classes.

Open Decisions — TBD

TBD-1 · Safety Seconds semantics for FX / CF / XF Parked

Safety Seconds currently means "wait N seconds after market open before Move-SL-to-Cost activates" — designed around NSE's 09:15 open.

Problem: FX, commodity, and crypto trade nearly continuously. No clean "open" to anchor against.

Proposed direction: anchor Safety Seconds to position entry time — "ignore the first N seconds after this leg entered." Applies uniformly to FX, CF, XF.

Status: parked.

TBD-2 · Tick-size enforcement for SL / Target trigger prices Parked

Engine currently does not enforce per-instrument tick on user-entered SL/Target values.

  • (a) Accept as-is
  • (b) Snap silently on save
  • (c) Reject on save

Recommended: (b) — snap silently on save, show snapped value in UI.

Status: parked — recommendation noted, decision pending.

Summary by asset class

Index Futures (IF) — Indian + US

  • Keep: Combined Loss, Underlying Movement, Loss and Underlying Range.
  • Adapt: SameStrike → "Same Contract"; per-instrument lot_size.
  • PnL currency: INR (Indian) / USD (US).

FX Futures (FX)

  • Drop: Underlying Movement, Loss and Underlying Range.
  • Adapt: per-instrument lot_size; Safety Seconds per TBD-1.
  • PnL currency: USD.

Commodity Futures (CF)

  • Same drops as FX.
  • Adapt: per-instrument lot_size and tick size; Safety Seconds per TBD-1.
  • PnL currency: USD (MCX physical INR normalized to USD).

Crypto (XF) — Futures + Spot

  • Same drops as FX.
  • Adapt: per-instrument lot_size (futures); exchange minimum (spot); Safety Seconds per TBD-1.
  • PnL currency: USD.
