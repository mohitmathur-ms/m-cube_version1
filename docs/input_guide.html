Crypto Data App — CSV Input Guide

A beginner-friendly guide to preparing your cryptocurrency data files for the dashboard. No coding experience needed.

What Does This App Do?

This app lets you load cryptocurrency price history from CSV files (spreadsheet files) and then:

  • View interactive price charts
  • Run trading strategy backtests (test "what if I traded like this in the past?")
  • Compare different coins side by side

To do all of this, the app needs your data in a specific format. This guide explains exactly what that format looks like — think of it as a recipe the app expects you to follow.

Quick Start — Your First CSV in 3 Steps

If you just want to get started fast, follow these three steps:

  1. Name your file correctly Use the pattern: ID_SYMBOL_Name.csv
    Example: 1_BTC_Bitcoin.csv
  2. Make sure it has these 6 column headers Your first row must be: ts,open,high,low,close,volume
    Exactly those names, exactly that lowercase spelling.
  3. Load it in the app Open the app, go to the "Load Data" page, point it to your CSV folder, select your file, and click "Load Selected into Catalog".
Tip The app already has quick-select buttons for popular coins like BTC, ETH, SOL, XRP, DOGE, and more — so if your files are named correctly, loading is just one click!

How to Name Your File

The app scans your folder and looks for files named in a very specific way. If the filename doesn't match, the file is invisible to the app (no error, it just won't show up).

The Pattern

{number}_{SYMBOL}_{Name}.csv
Part What It Means Rules Example
{number} A numeric ID for the coin (from CoinMarketCap) Digits only (1, 42, 1027, etc.) 1
{SYMBOL} The coin's ticker symbol UPPERCASE letters only — no numbers, no lowercase BTC
{Name} The coin's full name (human-readable) Any text is fine Bitcoin

Each part is separated by an underscore _ and the file must end with .csv.

Correct Names

1_BTC_Bitcoin.csv
1027_ETH_Ethereum.csv
5426_SOL_Solana.csv
74_DOGE_Dogecoin.csv
52_XRP_XRP.csv

Incorrect Names — Won't Load

BTC_Bitcoin.csv — missing the ID number
1_btc_Bitcoin.csv — "btc" must be "BTC"
bitcoin_data.csv — doesn't follow the pattern
1_BTC_Bitcoin.xlsx — must be .csv, not .xlsx
1-BTC-Bitcoin.csv — uses hyphens instead of underscores
Where do I find the coin ID? Go to CoinMarketCap.com and look at the URL of any coin's page. For example, Bitcoin's URL is coinmarketcap.com/currencies/bitcoin/ and its ID is 1. You can also find it in the CoinMarketCap API. If you're unsure, just use any unique number.

The 6 Columns Your CSV Must Have

Think of each row in your CSV as a snapshot of a coin's price for one day (called a "bar" or "candle" in trading). Here's what each column means:

Column Name What It Means (Plain English) What Kind of Data Example
ts required The date (or date+time) this price data is for A date, datetime, or number representing time 2024-01-15
open required The price when trading started that day A number (decimals OK) 42265.19
high required The highest price reached that day A number (decimals OK) 45000.00
low required The lowest price reached that day A number (decimals OK) 41800.50
close required The price when trading ended that day A number (decimals OK) 44172.65
volume required How much of the coin was traded that day A number (can be very large) 25483910000
Column Names Are Case-Sensitive! ts works. Ts, TS, Timestamp — none of these will work. The column names must be exactly lowercase as shown above.
Have extra columns? That's perfectly fine! If your CSV has additional columns (like market_cap, adj_close, etc.), the app will simply ignore them. Only the 6 columns above are used.

Timestamps — Date/Time Formats Explained

The ts column is the most flexible column. Here are all the date/time formats that will work:

Format Example Works?
Standard date (recommended) 2024-01-15 Yes
Date with time 2024-01-15T00:00:00 Yes
Date with timezone info 2024-01-15T00:00:00+00:00 Yes
Unix timestamp (seconds since 1970) 1705276800 Yes
Unix timestamp in milliseconds 1705276800000 Yes
Slash-separated (US style) 01/15/2024 Risky — month/day can be ambiguous
Written-out date Jan 15, 2024 Yes
Random text / empty cell hello or blank Error
What is UTC? UTC is the global standard time (like GMT). If your dates don't specify a timezone, the app assumes they're in UTC. If your data is from a source in a different timezone (say, EST), your chart might be off by a few hours — usually not a problem for daily data.

What Happens After You Load a CSV

When you click "Load", the app processes your file through 7 steps. Here's what happens behind the scenes in plain English:

  1. Read the file The app opens your CSV file and reads all the rows and columns.
  2. Check that all 6 columns exist It looks for ts, open, high, low, close, volume. If any are missing, it stops and shows an error message telling you which ones are missing.
  3. Parse the dates The ts column is converted into proper date/time values. All dates are treated as UTC time.
  4. Convert prices & volume to numbers The app tries to turn every value in the price and volume columns into a number. If a value can't be converted (e.g., it's text like "N/A"), it gets marked as "missing".
  5. Remove incomplete rows Any row that has a missing value in any column gets removed entirely. (This happens silently — you won't see a warning.)
  6. Sort by date All rows are arranged in chronological order (oldest date first).
  7. Cap very large volume numbers If any volume value is above 16 billion, it's reduced to 16 billion. This is a safety limit to prevent crashes in the trading engine.

Tricky Situations & What the App Does

Things the App Fixes Silently (No Error Shown)

These issues won't crash the app, but your data may be quietly modified:

  • ⊝
    A cell contains text instead of a number (e.g., "N/A" in the volume column)
    What happens: The entire row is removed. You won't see a warning or a count of how many rows were dropped.
  • ⊝
    A cell is empty / blank
    What happens: Same as above — the entire row is removed silently.
  • ⇄
    Volume is larger than 16 billion
    What happens: The value is capped at 16,000,000,000. For example, 20000000000 becomes 16000000000.
  • ⇅
    Rows are not in date order
    What happens: The app re-sorts all rows from oldest to newest automatically.
  • ✓
    CSV has extra columns (like market_cap, adj_close)
    What happens: Extra columns are ignored. Only the 6 required columns are kept.
  • ✓
    Dates don't include timezone info
    What happens: The app assumes all dates are in UTC. No warning is shown.

Things the App Allows (But You Should Watch Out For)

These won't cause errors, but could lead to weird results in charts and backtests:

  • !
    Negative prices (e.g., open = -100)
    Allowed but wrong: Crypto prices should never be negative. The app doesn't check for this.
  • !
    High price is lower than Low price
    Allowed but wrong: Logically, high should always be ≥ low. The app doesn't validate this.
  • !
    Duplicate dates (two rows with the same timestamp)
    Allowed: Both rows are kept. This could distort your backtest results.
  • !
    Completely identical rows
    Allowed: Not detected or removed. All copies are kept.
  • !
    Empty file (only headers, no data rows)
    Allowed: Loads without error, but there will be nothing to chart or backtest.

Mistake 2: Non-numeric values in price columns

What a problematic CSV looks like
tsopenhighlowclosevolume
2024-01-0142265.1945000.0041800.5044172.6525483910000
2024-01-02$44,172.6544950.0043200.0043850.10N/A

The highlighted row has $44,172.65 (with a dollar sign and comma) and N/A. This row will be silently removed because those aren't pure numbers.

Rule of thumb Numbers must be plain numbers only. No dollar signs ($), no commas (44,172), no percentage signs (%), no text (N/A, null, -).

Mistake 3: Wrong filename format

If your file doesn't appear in the app's list, double-check the filename. The most common issues:

  • Using lowercase symbol: 1_btc_Bitcoin.csv → Fix: 1_BTC_Bitcoin.csv
  • Missing the ID number: BTC_Bitcoin.csv → Fix: 1_BTC_Bitcoin.csv
  • Using the wrong separator: 1-BTC-Bitcoin.csv → Fix: use underscores _
  • Wrong file extension: 1_BTC_Bitcoin.xlsx → Fix: save/export as .csv

Error Messages Explained

If something goes wrong, here's what the error messages mean and how to fix them:

Error Message What It Means How to Fix It
CSV missing required columns: ['volume'] Your file doesn't have one or more of the 6 required columns. The error tells you which ones are missing. Add the missing column(s) to your CSV. Check spelling — it's case-sensitive.
FileNotFoundError The app can't find the file at the path you provided. Double-check the folder path in the app. Make sure the CSV file is actually in that folder.
Unknown datetime string format The ts column has dates in a format the app can't understand. Convert your dates to a standard format like 2024-01-15 (YYYY-MM-DD).
Out of bounds nanosecond timestamp A date is too far in the past or future (outside the year range ~1677 to ~2262). Check for obviously wrong dates, like year 0001 or 9999.
"No CSV files found in {folder}" The folder exists but contains no files matching the naming pattern. Rename your files to match the ID_SYMBOL_Name.csv pattern.

Pre-Upload Checklist

Go through this list before loading your CSV. If everything checks out, you're good to go!

  • File is named correctly — follows ID_SYMBOL_Name.csv pattern (e.g., 1_BTC_Bitcoin.csv)
  • File extension is .csv — not .xlsx, .txt, or .json
  • First row has all 6 column headers — ts, open, high, low, close, volume
  • Column names are exact lowercase — Open won't work, must be open
  • Dates are in a standard format — 2024-01-15 is the safest choice
  • Numbers are plain numbers — no $, no commas, no N/A, no blanks
  • Dates are in UTC (or you're OK with the app assuming UTC)
  • No obviously wrong data — no negative prices, no high < low values
  • No duplicate dates (unless intentional) — each row should be a unique day
  • Volume values are under 16 billion — larger values will be silently capped

Frequently Asked Questions

Can I use data exported from Excel or Google Sheets?
Yes! Just make sure you export/save as CSV (Comma Separated Values), not as .xlsx or .ods. In Excel: File → Save As → choose "CSV (Comma delimited)". In Google Sheets: File → Download → Comma Separated Values (.csv). Then rename the file to match the required pattern.
Where can I download crypto CSV data?
Popular free sources include:
  • CoinMarketCap — historical data section (may need free account)
  • Yahoo Finance — search for a crypto, click "Historical Data", then "Download"
  • CoinGecko — export historical data from any coin's page
  • Kaggle — search for "cryptocurrency historical data" datasets

After downloading, you may need to rename the columns to match what this app expects (ts, open, high, low, close, volume) and rename the file to match the naming pattern.

My file doesn't show up in the app. What's wrong?
Three things to check:
  1. Is the file in the right folder? Check the folder path shown in the app.
  2. Is the filename correct? It must match ID_SYMBOL_Name.csv exactly — uppercase symbol, underscores as separators, .csv extension.
  3. Is it actually a .csv file? Sometimes files look like .csv in Windows but are actually .csv.txt. Enable "Show file extensions" in your file explorer to verify.
Some of my rows disappeared after loading. Why?
The app silently removes any row where any value is missing, blank, or can't be converted to a number. Open your CSV in a spreadsheet and look for:
  • Blank cells
  • Text values like "N/A", "null", "-", or "#REF!"
  • Numbers with formatting: "$1,234" or "1,234.56" (commas aren't allowed)

Fix those cells and reload.

Does the data have to be daily? Can I use hourly or minute data?
The CSV loader itself doesn't care about the time interval — it just reads rows with timestamps. However, the backtest engine processes the data as daily bars, so hourly or minute data will be treated as one bar per row regardless. For best results, use daily OHLCV data (one row per day).
Can I load multiple coins at once?
Yes! Place all your CSV files in the same folder, and the app will detect all of them. You can then select multiple coins from the list or use the quick-select buttons. Each coin needs its own separate CSV file.
What does "CoinMarketCap ID" mean? Does it matter what number I use?
The ID is just a unique number to identify each coin. CoinMarketCap assigns these (Bitcoin = 1, Ethereum = 1027, etc.), but any unique number will work. The app uses it mainly to sort and identify files. If you don't know the official ID, just pick any number — it won't affect your charts or backtests.
My data uses different column names (like "Date", "Open", "Close"). Do I have to rename them?
Yes. The app requires exact column names: ts, open, high, low, close, volume. If your data has Date, rename it to ts. If it has Open, rename it to open. You can do this in Excel/Google Sheets by simply editing the first row, or in a text editor by changing the first line.
