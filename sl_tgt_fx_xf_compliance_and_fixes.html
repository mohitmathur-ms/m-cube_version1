UI policy for options/live-only values

All spec values stay visible in the UI; values that don't apply to FX/XF (options-only / IF-only / live-only) are kept as selectable options but marked gray with a "(options only)" or "(live/options only)" suffix and the pf-live-only CSS class. The backend silently downgrades these to the FX/XF universal value with a warning log — choosing a grayed value won't crash a backtest.

Backend verification

No backend code changes were required. Every newly-exposed field already existed end-to-end:
